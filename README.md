# Desafio de Projeto: Refinando um Modelo Conceitual de E-commerce

Este repositório contém a modelagem de dados para um sistema de E-commerce, focando na especialização de clientes e gestão de entregas.

## 📌 Requisitos Atendidos

- **Clientes PF e PJ:** O modelo suporta ambos os tipos de clientes, garantindo que uma conta não possua ambas as informações simultaneamente.
- **Múltiplos Pagamentos:** O cliente pode cadastrar diversas formas de pagamento para facilitar suas compras.
- **Rastreamento de Entregas:** Implementação de controle de logística com status de envio e código de rastreio para o consumidor.

## 🛠️ Ferramentas Utilizadas
- MySQL Workbench (ou ferramenta de sua preferência)
- Notação EER (Enhanced Entity-Relationship)
